Journey & Implementation Story

This project started as a deep dive into how network traffic actually moves through a system. I wanted to move beyond just reading logs and instead see the raw packets as they hit the wire.

The biggest challenge was loopback sniffing. On many Linux distributions, standard Layer 3 sockets miss traffic sent to localhost. I had to pivot to using Scapy's Layer 2 sendp() and sniff() logic, which required a much deeper understanding of Ethernet frames and interface indices.

The WAF evolved from a simple set of regex strings to a contextual inspector. I learned early on that infrastructure headers (like Host or Content-Type) often contain words like "DELETE" or "UPDATE" that would trigger false positives. The current implementation uses a whitelist approach, only scanning user-controlled inputs to ensure high fidelity detection.

Technical Architecture

NIDS Engine (Port 5000)

Uses Scapy for asynchronous packet capture. It analyzes TCP flags, ICMP types, and payload patterns to detect scans and floods.

WAF/IDS (Port 5001)

A Flask-based inspector that validates HTTP request bodies, paths, and headers against injection patterns.

Both servers emit events via Socket.IO to a React frontend, providing a real-time "Threat Feed" that updates without page refreshes.

Setup Guide

Critical Security Note: The NIDS engine requires sudo (root) privileges to open raw sockets for network sniffing. In the Docker environment, this is handled by privileged: true and network_mode: host.

Local Startup

# Install dependencies
pip install -r backend/requirements.txt

# Start NIDS (Requires Sudo)
sudo python backend/app.py

# Start WAF
python backend/waf_app.py

Docker Deployment

docker-compose up --build

Demonstration & Validation

The Attack Simulator allows for safe, controlled testing of NIDS rules. When a "SYN Flood" is triggered, the engine tracks the rapid succession of SYN packets and emits a "Critical" alert once the threshold is crossed.

The WAF's robustness is backed by over 100 test cases in test_waf.py, covering edge cases for SQL injection, XSS, and path traversal across various encoding formats.
